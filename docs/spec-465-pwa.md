# 衆議院議員465名 単語帳 PWA 仕様

## 目的
- 衆議院議員465名を、顔写真と氏名の対応を崩さずに学習できる iPhone 優先の単語帳アプリとして運用する
- 小選挙区、比例代表、検索の3導線で迷わず辿れる構成にする
- 既存の JSON と写真データを正規ソースのまま維持し、将来の差し替えや追加を安全にする

## 対象端末
- 最優先: iPhone Safari
- 次点: iPad Safari、Android Chrome、PC ブラウザ
- PWA としてホーム画面追加された状態を正式サポート対象に含める

## データ構造
### 正規ソース
- 一覧入口: `data/members/index.json`
- 個票: `data/members/hr-XXXX.json`
- 写真: `data/photos/hr-XXXX.jpg`
- 元資料: `members.pdf` と `data/source-pdf/members.pdf`
- バッチ台帳: `data/batches/*`

### アプリ内の派生項目
- 既存 JSON は `electionType` / `district` / `block` / `photo` を保持する
- UI 側では破壊的移行を避けるため、以下を派生項目として生成する
- `districtType`: `electionType` の別名
- `districtName`: `district` の UI 用別名
- `proportionalBlock`: `block` の UI 用別名
- `image`: `photo` の UI 用別名
- `districtOrBlockLabel`: 区分に応じた表示用ラベル
- `searchText`: 検索用の連結文字列

### データ保全方針
- 元 JSON の必須キーは削除しない
- 氏名と写真の対応を変える変更は、必ず個票 JSON と実画像を対で確認する
- スキーマ拡張が必要でも、まずは派生項目で吸収し、元データの一括書き換えは急がない

## 画面構成
### 共通
- 上部に全体概要と収録数を表示する
- PWA 追加案内を表示できる
- タブは `小選挙区` `比例代表` `検索` の3つ
- 右または上に単語帳カード、左または下に一覧を置く

### 小選挙区タブ
- 都道府県セレクトを表示する
- 選択都道府県の選挙区一覧をチップ表示する
- 顔写真、氏名、選挙区が見えるカード一覧を出す

### 比例代表タブ
- 比例ブロックセレクトを表示する
- 顔写真、氏名、比例ブロックが見えるカード一覧を出す

### 検索タブ
- 氏名検索
- 政党絞り込み
- 都道府県絞り込み
- 小選挙区 / 比例代表の切り替え絞り込み
- 0件時の空状態表示
- 初期表示は分割読み込みし、全件一括描画を避ける

## 機能一覧
- 単語帳カードの表裏切り替え
- 前へ / 次へ移動
- 小選挙区の都道府県別閲覧
- 比例代表のブロック別閲覧
- 氏名検索と複合絞り込み
- 直前のタブ、選択議員、主な絞り込み条件の保持
- PWA 追加案内
- Service Worker による最低限のオフライン利用
- 簡易監査スクリプトによる重複・参照確認

## カード仕様
### 表面
- 顔写真
- 氏名
- 選挙区または比例ブロック
- 区分ラベル
- 政党名

### 裏面
- 所属政党
- 都道府県
- 区分
- 選挙区または比例ブロック
- 当選回数
- 主な経歴
- 補足情報

### 安定性方針
- 表裏切り替えは CSS の表示切り替えで実装し、重い 3D アニメーションは使わない
- 写真は `object-fit: contain` を基本にして顔の欠けを防ぐ

## 操作フロー
1. アプリ起動後、直前のタブまたは既定タブで一覧を見る
2. 一覧から議員を選ぶ
3. 表面で顔写真と氏名、選挙区または比例情報を確認する
4. 裏面で政党、都道府県、区分、補助情報を確認する
5. 前後移動または別タブに切り替えて学習を続ける

## 今後の修正方針
- 既存データは `data/` を正規ソースとして維持する
- UI 上の派生項目追加は `src/member-store.js` で吸収する
- データ修正時は `data/members/hr-XXXX.json` と `data/photos/hr-XXXX.jpg` を同時に確認する
- バッチ追加時は `index.json`、個票 JSON、写真、バッチ台帳を同期する
- 仕様変更が入る場合はこの文書か README に判断理由を残す

## パフォーマンス方針
- 一覧画像は `loading="lazy"` を使う
- 検索結果は段階表示し、iPhone での一括描画を抑える
- 詳細 JSON は必要時読み込みし、選択済みデータはメモリキャッシュする
- Service Worker はアプリ殻と一覧入口を事前キャッシュし、詳細 JSON と画像は実行時キャッシュする
- 過剰なアニメーションや仮想化は導入しない

## 検品項目
- 小選挙区 / 比例代表 / 検索 の3タブが明確
- iPhone 幅でタップしやすい
- 写真と氏名の対応が正しい
- 小選挙区は都道府県から選挙区へ辿れる
- 比例代表はブロック単位で辿れる
- 検索で 0 件時に分かりやすい表示になる
- ホーム画面追加が案内される
- オフライン時に少なくともアプリ殻と既読データが使える
- `npm test` `npm run validate` `npm run audit` の結果を確認できる

## 既存データを壊さないルール
- `name` と `photo` の対応を変える変更は禁止
- 個票 JSON の `id` とファイル名は一致させる
- `photo` は `/data/photos/hr-XXXX.jpg` を維持する
- `memberPath` は `/data/members/hr-XXXX.json` を維持する
- `electionType` と `district` / `block` の整合性を保つ
- 大規模な一括置換を行う前に、必ず 1 件サンプルで表示確認する

## 今回の判断メモ
- `districtType` などの新項目は元 JSON に直接追加せず、コード上の派生項目として扱う
- 理由は、465件の既存データを破壊的に移行せず、今後も差分で追いやすくするため
- PWA のオフライン対応は「最低限使える」範囲に留め、初回から全画像を事前キャッシュしない
- 理由は、iPhone でのストレージ負荷と初回通信量を抑えるため
