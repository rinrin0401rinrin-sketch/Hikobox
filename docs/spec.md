# 衆議院議員単語帳アプリ 仕様メモ

## 目的
- 衆議院議員465名を顔写真つき単語帳UIで学習できるようにする
- いきなり全件実装せず、完成見本1名を基準に安全に拡張する
- PDFは参照元にとどめ、正規データは `data/` 配下の JSON で管理する

## 今回の作業範囲
- 完成見本1名のデータ構造を確定する
- JSON 1件を読み込む単語帳カードUIを実装する
- 一覧画面と絞り込みUIの土台を作る
- 20名・50名追加に耐える命名規則と検証スクリプトを整備する

## 非目標
- 465名分の一括量産
- PDF からの自動大量抽出
- 学習済み、苦手登録、ランダムの本実装
- 管理画面や CMS の構築

## データの正規配置
- メンバー詳細: `data/members/hr-0001.json`
- メンバー一覧: `data/members/index.json`
- 写真: `data/photos/hr-0001.jpg`
- バッチ管理: `data/batches/*/manifest.json`

## 画面構成
### 学習画面
- 単語帳カードの表面: 顔写真、氏名、政党
- 単語帳カードの裏面: 選出区分、選挙区または比例ブロック、当選回数、主な経歴、補足メモ
- 操作: 前へ、次へ、めくる
- 将来拡張用のプレースホルダー: ランダム、学習済み、苦手登録

### 一覧画面
- 写真サムネイル
- 氏名
- 政党
- 選出区分
- 詳細カードへ遷移する導線

### 絞り込み
- 政党
- 選出区分
- 都道府県
- 比例ブロック

## JSON スキーマ
```json
{
  "id": "hr-0001",
  "name": "氏名",
  "nameKana": "",
  "party": "政党名",
  "electionType": "single",
  "district": "",
  "block": "",
  "prefecture": "",
  "wins": 0,
  "birthDate": "",
  "age": null,
  "career": [],
  "photo": "/data/photos/hr-0001.jpg",
  "sourcePdf": "",
  "sourcePage": null,
  "status": "draft",
  "notes": ""
}
```

## index.json の役割
- 一覧画面用の軽量データをまとめる
- 各議員 JSON への `memberPath` を持たせる
- フィルタに必要な最小項目を先に読めるようにする

## バッチ方針
- `batch-00-sample`: 完成見本1名
- `batch-01-20`: 検証用20名
- `batch-02-50`: 50名追加の最初の標準バッチ
- 以降は50名単位を基本にし、端数は別バッチに分ける

## 現時点の制約
- ワークスペース内に参照 PDF と正式写真が未配置
- そのため `hr-0001` は UI・スキーマ検証用の `draft` 完成見本として作成
- 実データ投入時は `sourcePdf` `sourcePage` `party` `district/block` `career` を裏取りして更新する
