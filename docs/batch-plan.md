# バッチ追加計画

## batch-00-sample
- 目的: 完成見本1名でデータ構造、UI、検証スクリプトを固める
- 件数: 1名
- 完了条件:
  - JSON と写真が紐づく
  - 単語帳 UI が成立する
  - validate が通る

## batch-01-20
- 目的: 表記ルールと運用ルールを20名で検証する
- 件数: 20名
- 進め方:
  - 完成見本 `hr-0001` 青山繁晴を基準カードにする
  - まず同型の小選挙区基本型グループから着手する
  - 次に小選挙区変動型、比例代表型、保留・例外型の順で広げる
  - 党派、選出区分、都道府県、比例ブロックの表記を揃える
  - 20名投入のたびに validate を回す
  - UI 一覧と絞り込みで破綻がないか確認する
- 完了条件:
  - 表記ゆれの修正方針が固まる
  - 画像配置と JSON 命名規則が崩れない
  - 青山繁晴基準のグループ順で追加しても差分運用が破綻しない

## batch-02-50
- 目的: 量産の最初の標準バッチを確立する
- 件数: 50名
- 前提:
  - batch-01-20 でルール固定済み
  - バリデーション項目が不足なく機能している
- 完了条件:
  - 追加フローが機械的に繰り返せる
  - レビュー時に差分追跡しやすい

## batch-03 以降
- 基本は50名ずつ追加
- 最終端数は別バッチに分ける
- 各バッチでやること:
  - JSON 追加
  - 写真追加
  - index 反映
  - manifest 更新
  - validate 実行
