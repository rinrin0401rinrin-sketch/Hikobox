# バッチ追加計画

## batch-00-sample
- 目的: 青山繁晴の完成見本1名で、データ構造、UI、検証スクリプトの基準を固定する
- 件数: 1名
- 状態: 完了
- 完了条件:
  - JSON と写真が紐づく
  - 出典 PDF とページ番号が記録されている
  - 単語帳 UI が成立する
  - validate が通る

## batch-01-20
- 目的: 青山繁晴の完成見本を基準に、20名で表記ルールと運用ルールを検証する
- 件数: 20名
- 進め方:
  - 政党、選出区分、都道府県、比例ブロックの表記をそろえる
  - 各人に写真と `sourcePdf` `sourcePage` を付ける
  - 追加のたびに validate を回す
  - UI 一覧と絞り込みで破綻がないか確認する
- 完了条件:
  - 表記ゆれの修正方針が固まる
  - 画像配置と JSON 命名規則が崩れない
  - 完成見本を見ずに同じ粒度で追加できる

## batch-02-50
- 目的: 20名検証で固まったルールを使って、50名追加の標準手順を確立する
- 件数: 50名
- 前提:
  - batch-01-20 でルール固定済み
  - バリデーション項目が不足なく機能している
- 完了条件:
  - 追加フローが機械的に繰り返せる
  - レビュー時に差分追跡しやすい

## batch-03 以降
- 基本は50名ずつ追加
- 最終端数は別バッチに分ける
- 各バッチでやること:
  - JSON 追加
  - 写真追加
  - `index.json` 反映
  - manifest 更新
  - validate 実行

## 次段階の具体タスク
1. batch-01-20 の候補20名を選ぶ
2. 20名分の写真と PDF ページ対応表を用意する
3. 青山繁晴の完成見本と同じ粒度で 20 名を投入する
4. 必要なら `index.json` 自動生成スクリプトを追加する
